# BoboLucyWisdom 维基结构规范

## 领域
健康、养生、神经科学、营养学、生物黑客和长寿洞察 —— 将 @BoboLucyWisdom 的内容组织成结构化维基。

## 规范
- 文件名：小写，连字符，无空格（例如 `intermittent-fasting.md`）
- 每个维基页面以 YAML 前言开始
- 使用 `[内部链接](内部链接)` 在页面间链接（每页至少2个出站链接）
- 更新页面时，务必更新 `updated` 日期
- 每个新页面必须添加到 `index.md`
- 每个操作必须追加到 `log.md`

## 前言
```yaml
---
title: 页面标题
created: YYYY-MM-DD
updated: YYYY-MM-DD
type: entity | concept | comparison | query | summary | theme
source_post: [tweet_filename.md]
tags: [来自下方分类]
---
```

## 标签分类

### 健康领域
- `sleep` — 睡眠优化、昼夜节律
- `nutrition` — 饮食、补剂、禁食
- `fitness` — 运动、活动、体能训练
- `mental-health` — 大脑健康、压力、焦虑
- `longevity` — 衰老、寿命、健康寿命
- `neuroscience` — 脑科学、认知
- `biochemistry` — 细胞过程、代谢

### 内容类型
- `biohacking` — 自我实验、优化方案
- `medical-research` — 研究、试验、临床证据
- `supplements` — 维生素、矿物质、促智药
- `lifestyle` — 日常习惯、例行公事、实践
- `diet` — 特定饮食模式（生酮、地中海等）

### 实体
- `person` — 被提及的研究者、医生、专家
- `organization` — 大学、实验室、公司
- `study` — 具体研究论文或试验
- `bio-marker` — 可测量的健康指标

### 元数据
- `concept` — 核心概念和框架
- `comparison` — 并排分析
- `protocol` — 分步骤方法
- `myth` — 已被证伪的 claims

## 页面阈值
- **创建页面** 当健康主题出现在2+帖子中，或在一篇帖子中是核心内容
- **添加到现有页面** 当帖子提及已涵盖的内容
- **不创建页面** 对于顺带提及、次要细节
- **拆分页面** 当它超过约200行
- **归档页面** 当被取代时 — 移动到 `_archive/`

## 实体页面
每个值得注意的实体一个页面。包括：
- 概述 / 是什么
- 关键事实、资历、所属机构
- 与其他实体的关系（[内部链接](内部链接)）
- 来源参考（帖子链接）

## 概念页面
每个概念或主题一个页面。包括：
- 定义 / 解释
- 当前知识状态
- 证据质量（引用的研究）
- 开放问题或争论
- 相关概念（[内部链接](内部链接)）

## 对比页面
并排分析。包括：
- 对比的内容及原因
- 对比维度（优先使用表格格式）
- 基于证据的 verdict 或综合
- 来源

## 方案页面
分步健康方法。包括：
- 先决条件 / 适用人群
- 方案（编号步骤）
- 预期结果 / 时间线
- 风险 / 禁忌症
- 参考

## 更新政策
当新信息与现有内容冲突时：
1. 检查日期 — 较新的来源通常取代较早的
2. 如果确实存在矛盾，用日期/来源注明两种立场
3. 在前言中标记矛盾：`contradictions: [page-name]`
4. 在 lint 报告中标记供用户审核
